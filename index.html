Description

This is open source filesystem library. It is currently tested on Windows Server 2019, Ubuntu 18.04/20.04, macOS 10.15. Cross-compiling is not possible at this time because there are so many OS-specific build tasks. This library provides only file system control, so you have to implement the abstraction layer for the library. However, in this regard, it can have the advantage of being able to integrate into the Operating system.

In this image, the disk device abstraction layer is in user application block. But if I transform this image a little bit...

You can integrate the library to your operating system!

But currently this repository is in dev, it is not thread-safe class. There's no disk scheduler. You can only create single thread for single filesystem.
